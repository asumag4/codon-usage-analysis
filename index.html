Introduction

This is a comprhensive overview of codon usage frequencies in the genomic coding DNA, tabulated in CUTG database (RNA). Included are codon frequencies from 514 genus, of which 13028 differing DNA species were found. This project aims to explore the dataset to derive biological conclusions. As well as attempt to create a predictive model; from codon characteristics, can we predict the Kingdom or the DNA type that the sequence originated to?

Data Set & Previous Work

This dataset has been published in Nature - Scientific Reports in Feb 2023 by Hallee & Khomtchouk. The dataset was modelled by the authors, and we'll compare my proposed model with the published one to identify the steps taken and if possible, take a different appraoch but reach similar levels of accuracies.

Codons Explained

Next, we'll go cover some domain knowledge so the reader is able to follow through the analyses and connect the insights draw out from the data. We'll first compare how data is encoded through computers vs. biology. In computers, we use binary code. In the modern world, it's covered by 64 bits, but I'll go over 8 bit binary code. For example; the 8 bit binary encoding for the word "Apple" is 01100001 01110000 01110000 01101100 01100101 , strung together. Now biology works in a similar fashion; Central Dogma of Biology

  1. DNA: Long-term (genetic material inherited from our parents) information that determines who we become. This is like the source code of a program. It contains all the instructions and data necessary for the program to function but isn't actively running the processes.
  2. mRNA: RAM Data (mRNA): Think of this as the compiled bytecode or an intermediate representation of the program. It takes the essential parts of the source code (genes) and brings them to the CPU (ribosomes) for execution.
  3. Polypeptide: when folded, it becomes an active machine called a **Protein** that enables cellular active and whats makes/ keeps you alive. It's synonymous to the executable program, once the program is running, it performs various functions, analogous to how proteins carry out various functions in a cell.
We'll be studying mRNA level of code, and triplets of each unit of data (RNA base) map to an amino acid. [For example; AUG maps to methionine; an amino acid usually used at the start of translation.] And a string of amino acids (when folded) make a protein!

Chemical Phases

Let's just review quickly about chemical character; **hydrophobicity** (does not mix with water) and **hydrophilicity** (does mix with water). I'm introducing this concept because amino acids hold either a hydrophobic or hydrophilic character. *When an amino acid is hydrophilic*, when it is part of a protein, it will associate well with water, and will usually be found on the outside of the protein, interacting with water molecules. *When an amino acid is hydrophobic* it will usually make up the inside of the protein, because it wants to hide and dissociate with water molecules, it will let hydrophillic amino acids do that. Apart from making the insides of a protein, hydrophobic amino acids can make up the "outside" of membrane associating proteins. Membranes "phospholipid bilaryer" look this:

and make up the "walls" of a cell! This is the core structure of what made & makes life possible! That's it for now... I will continuously refer back to this section as I draw insights in the data!

Conclusions

Throughout the analysis we find the profiles of genomic versus non-genomic DNA. We find the key hallmarks of what makes life possible on Earth, specifically oxygen-dependent species within the mammalian kingdom. We find data pertaining to mitochondria to be of particular interest for it's variation. We also determine the most and least frequently used codons throughout the kingdoms of life, and how they pertain to function and efficiency. We find patterns in the size of data that life uses and despite their size, restrict the usage of specific codons to specialize the chemical activity within cells. Lastly, we attempt to model the data in hopes to replicate the amazing performance from Hallee & Khomtchouk, where they train a model with 96% accuracy for kingdoms and 99% accuracy for DNA types. The difference between the model determined in this study and the published model is that theirs uses a simple hard-voting ensemble of k-NN, SVM and RF classifiers, whereas ours uses only RF. Further steps to optimize the model would be (1) Balance the dataset (2) Standardize the dataset (3) Implement a hard-voting system, and experiment with a soft-voting system as well. Further algorithm would be required. Thank you for exploring this dataset with me!
